## Overview

The **fhirEMSCore** Implementation Guide (IG) defines a set of FHIR R4 profiles, extensions, value sets, and code systems for Emergency Medical Services (EMS) Patient Care Report (PCR) data. This IG maps the **NEMSIS Version 3.5.1.251001CP2** data standard to FHIR R4, aligned with **US Core 6.1.0**.

The project demonstrates how NEMSIS — the US national EMS data standard — could be represented in FHIR to enable interoperability with EHR systems, health information exchanges, and public health infrastructure.

## Purpose

EMS agencies generate rich clinical and operational data during emergency responses. The NEMSIS standard has been the national EMS data standard since 2003 and defines hundreds of discrete data elements across multiple datasets. This IG explores bridging the gap between the NEMSIS XML-based data model and the FHIR interoperability ecosystem, which could enable:

- Exchange of EMS PCR data with hospital systems, HIEs, and public health agencies
- SMART on FHIR app integration for EMS workflows
- CMS and ONC compliance alignment
- Quality measure reporting and analytics

## Scope

This IG covers the following NEMSIS datasets:

| Dataset | Description | Primary FHIR Resources |
|---|---|---|
| EMSDataSet | EMS Patient Care Report | Composition, Encounter, Patient, Observation, Condition, Procedure, MedicationAdministration |
| DEMDataSet | Agency Demographics | Organization, Location, Practitioner |
| StateDataSet | State Configuration | Not directly profiled; informs value sets and extensions |
